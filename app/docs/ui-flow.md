# 빌드업 (Build-Up) — UI 화면 흐름도

**버전:** 0.1.0
**업데이트:** 2026-03-12

---

## 전체 흐름 개요

```
[최초 접속]
    │
    ▼
[/ 루트]
    │
    ├── Supabase 미설정 ──────────────────► [/setup] 설정 안내 페이지
    │
    ├── 비로그인 상태 ────────────────────► [/auth/login] 로그인
    │
    └── 로그인 상태 ──────────────────────► [/dashboard] 대시보드
```

---

## 인증 플로우

```
[/auth/login] 로그인
    │
    ├── 이메일 + 비밀번호 입력
    │       │
    │       └── 성공 ──────────────────────► [/dashboard]
    │
    └── 계정 없음 링크 클릭 ──────────────► [/auth/signup] 회원가입
            │
            ├── 이름 + 이메일 + 비밀번호 입력
            │       │
            │       └── 성공 → 이메일 인증 발송 → [이메일 확인 안내 화면]
            │                       │
            │                       └── 인증 링크 클릭 → [/auth/callback] → [/dashboard]
            │
            └── 이미 계정 있음 링크 ────────► [/auth/login]
```

## 메인 앱 플로우

### 대시보드 (`/dashboard`)

```
[/dashboard]
    │
    ├── 글로벌 빌드 점수 (전체 진행률 평균)
    ├── 통계 카드 4개 (전체진행률 / 총프로젝트 / 진행중 / 완료)
    ├── 활동 히트맵 (26주 × 7일)
    │
    ├── [새 프로젝트] 버튼 ─────────────────► [/projects/new]
    ├── [전체 보기] 링크 ───────────────────► [/projects]
    │
    └── 프로젝트 카드 클릭 ─────────────────► [/projects/:id]
```

### 프로젝트 목록 (`/projects`)

```
[/projects]
    │
    ├── 상태 필터 탭 (전체 / 기획 / 개발 / 테스트 / 완료)
    │
    ├── 프로젝트 카드
    │       ├── 이름, 상태 배지, 설명 (2줄)
    │       ├── 진행률 바
    │       ├── 멤버 수, 태스크 수, 최종 수정
    │       ├── 카드 클릭 ──────────────────► [/projects/:id]
    │       └── 삭제 버튼 (hover 시 표시) → 확인 다이얼로그 → 삭제
    │
    └── [새 프로젝트] 버튼 ─────────────────► [/projects/new]
```

### 새 프로젝트 생성 (`/projects/new`)

```
[/projects/new]
    │
    ├── 프로젝트 이름 (필수)
    ├── 설명 (선택)
    ├── 현재 단계 선택 (기획 / 개발 / 테스트 / 완료)
    │
    ├── [프로젝트 만들기] ──────────────────► [/projects/:id] (생성된 프로젝트)
    └── [← 프로젝트 목록] ──────────────────► [/projects]
```

### 프로젝트 상세 (`/projects/:id`)

```
[/projects/:id]
    │
    ├── 헤더
    │       ├── 프로젝트 이름 + 설명
    │       ├── [← 프로젝트 목록] ──────────► [/projects]
    │       └── [팀원] 버튼 ─────────────────► [/projects/:id/members]
    │
    ├── 왼쪽 (8/12) — 칸반 보드
    │       ├── [할 일] 컬럼
    │       │       ├── 태스크 카드 (드래그 가능)
    │       │       └── [+] 버튼 → 인라인 입력 → Enter로 추가
    │       ├── [진행 중] 컬럼
    │       │       └── (동일)
    │       └── [완료] 컬럼
    │               └── (동일)
    │               * 카드 드래그 → 다른 컬럼 드롭 → 상태 변경
    │               * 카드 hover → 삭제 버튼 표시
    │
    └── 오른쪽 (4/12) — 사이드패널
            ├── 프로젝트 상태 변경 (기획/개발/테스트/완료 버튼)
            ├── 진행률 슬라이더 + 저장
            ├── 피드백 & 코멘트
            │       ├── 입력창 + 전송 버튼
            │       └── 코멘트 목록 (최신순 20개)
            └── 활동 로그 (최신순 20개)
```

### 팀원 관리 (`/projects/:id/members`)

```
[/projects/:id/members]
    │
    ├── [← 프로젝트명] ──────────────────────► [/projects/:id]
    │
    ├── 팀원 초대 폼
    │       ├── 이메일 입력
    │       ├── 역할 선택 (관리자 / 멤버 / 테스터)
    │       └── [초대] → 이메일로 사용자 조회 → project_members 추가
    │
    └── 팀원 목록
            ├── 아바타, 이름, 이메일, 역할 배지
            └── 소유자 제외 → hover 시 [삭제] 버튼 표시
```

## 사이드바 내비게이션

```
[사이드바] — 모든 앱 페이지에 공통 표시
    │
    ├── 빌드업 로고 + 아이콘
    ├── [대시보드] ──────────────────────────► [/dashboard]
    ├── [프로젝트] ──────────────────────────► [/projects]
    └── [로그아웃] → Supabase signOut → [/auth/login]
```

---

## 칸반 보드 상세 인터랙션

```
태스크 카드
    │
    ├── 드래그 시작
    │       └── 카드 반투명 + 그림자 효과
    │
    ├── 다른 컬럼 위에서 드롭
    │       └── status 업데이트 (todo → in_progress → done)
    │
    ├── 같은 컬럼 내 드롭
    │       └── 현재: 상태 변경 없음 (추후 순서 변경 구현 예정)
    │
    ├── [+] 버튼 클릭
    │       ├── 인라인 텍스트 입력 표시
    │       ├── Enter → 태스크 생성
    │       └── Esc / 포커스 벗어남 → 취소
    │
    └── 카드 hover
            └── [삭제] 아이콘 표시 → 클릭 → 즉시 삭제
```

## 상태 배지 & 색상 규칙

| 상태 | 레이블 | 색상 |
|------|--------|------|
| `planning` | 기획 | 보라 (`purple-100 / purple-700`) |
| `development` | 개발 | 파랑 (`blue-100 / blue-700`) |
| `testing` | 테스트 | 노랑 (`yellow-100 / yellow-700`) |
| `completed` | 완료 | 초록 (`green-100 / green-700`) |

| 우선순위 | 레이블 | 색상 |
|----------|--------|------|
| `low` | 낮음 | 회색 (`slate-100 / slate-600`) |
| `medium` | 보통 | 주황 (`orange-100 / orange-600`) |
| `high` | 높음 | 빨강 (`red-100 / red-600`) |

| 역할 | 레이블 | 색상 |
|------|--------|------|
| `owner` | 소유자 | 주황 |
| `admin` | 관리자 | 파랑 |
| `member` | 멤버 | 회색 |
| `tester` | 테스터 | 보라 |
